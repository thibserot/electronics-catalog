# 99-S12 smart cabinet electromagnetic lock - AC005

Compact 12 V **fail‑secure** solenoid bolt used for cabinets, drawers and small doors. When powered, the bolt retracts (unlock). When unpowered, the spring pushes the bolt out (lock). Ideal when you need a simple latch you can pulse from an ESP32 via a MOSFET or relay—no position sensing built‑in.

## Specifications

Extracted from vendor listings for **99‑S12** (values vary by batch/variant):  

- **Operating voltage:** 12 V DC ("12 V universal")  
- **Current:** variants listed around **0.35 A / 0.6 A / 1.2 A** (instantaneous trigger) — measure your unit  
- **Pull stroke (throw):** **0–4 mm**  
- **Pull force:** at 4 mm ≥ **80 g** (typ. at 0.35 A)  
- **Retention:** at 0 mm ≥ **120 g**  
- **Operating mode:** power‑on = **unlock**; power‑off = **lock** (fail‑secure)  
- **Duty / interval note:** listing states *“power on 90%, interval”* — treat as **intermittent duty** (use short pulses)  
- **Ambient:** −25 °C to +80 °C; RH 45–80 %  
- **Withstand voltage:** AC 1000 V; **Insulation resistance:** 50 MΩ  
- **Leads:** 2‑wire pigtail  
- **Life:** up to **1,000,000** cycles (vendor‑stated)


## Pinout & Addresses (common breakout labels)

This part is **just the actuator**, no PCB:
- **Red (+12 V)** — connect to +12 V supply.  
- **Black (GND)** — connect to MOSFET/relay side to switch return to ground.  
(No logic pins, no addresses.)

## Wiring

**ESP32 cannot drive the lock directly.** Use a logic‑level N‑MOSFET or a relay module.

**MOSFET option (recommended for speed/size):**
- 12 V **+** → lock **red** wire.  
- Lock **black** wire → **DRAIN** of N‑MOSFET (e.g., IRLZ44N, AOZ/AOD logic‑level types).  
- **SOURCE** → 0 V (GND).  
- **GATE** → ESP32 GPIO via ~**100 Ω**; add **10 kΩ** gate pull‑down to GND.  
- **Flyback diode** across the lock (e.g., 1N5819/1N4007): **anode to MOSFET side**, **cathode to +12 V**.  
- **Common ground:** tie ESP32 GND to 12 V supply GND.

**Relay option:** ESP32 → Relay IN; relay switches 12 V to the lock. Still add a flyback diode across the coil if your relay board doesn’t have one.

## Gotchas

- Treat as **intermittent duty**: use short pulses (e.g., 100–500 ms) to unlock; avoid long holds or it will heat up.
- Current varies by batch (0.35 A–1.2 A). **Bench‑measure** your unit and size wiring/MOSFET accordingly.
- Always use a **flyback diode**; without it you risk brown‑outs or ESP32 resets.
- **Fail‑secure** behavior: loses power ⇒ **locks**. Plan for power‑loss scenarios.

## How to use (ESP32 / Arduino)

Simple pulse‑to‑unlock using a MOSFET (active‑HIGH gate). No external libraries needed.

```cpp
// GPIO driving a logic-level N-MOSFET gate through ~100Ω
const int LOCK_PIN = 16;  // choose any suitable GPIO

void setup() {
  pinMode(LOCK_PIN, OUTPUT);
  digitalWrite(LOCK_PIN, LOW); // ensure locked (no power to solenoid)
}

// Pulse to unlock for 200 ms
void unlockPulse(uint16_t ms = 200) {
  digitalWrite(LOCK_PIN, HIGH);  // power the solenoid => unlock
  delay(ms);
  digitalWrite(LOCK_PIN, LOW);   // remove power => lock
}

void loop() {
  // Example: unlock for 200 ms every 5 seconds
  unlockPulse(200);
  delay(5000);
}
```

If you use a **relay module** instead of a MOSFET, replace `LOCK_PIN` wiring accordingly (some boards are **active‑LOW**).
