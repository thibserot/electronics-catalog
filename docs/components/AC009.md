# SG92R 90-180 Degree - AC009

A tiny **9 g micro servo** (often sold as TowerPro SG92R) widely used in hobby robotics and small mechanisms. 
It rotates to a commanded angle using standard servo PWM (≈50 Hz) and provides modest torque for light linkages, knobs, and flaps. 
Use it when you need **position control** (~0–180°) rather than continuous rotation.

## Specifications

> AliExpress listing did not include a detailed table. The following are **typical** for SG92R; minor variations exist by vendor/batch—confirm in the datasheet or your specific listing.

- Operating voltage: **4.8–6.0 V** (5 V typical)  
- Stall torque (4.8 V): **~2.5 kg·cm**  
- Speed (4.8 V): **~0.1 s / 60°**  
- Mass: **~9 g**  
- Dimensions (body): **≈23 × 12.2 × 27 mm**  
- Gear train: **POM with carbon fiber**  
- Control: **50 Hz** servo signal (approx. **500–2400 µs** pulse range)  
- Typical current while moving: **100–300 mA**; **stall can reach ~650 mA**

## Pinout & Addresses (common breakout labels)

Color codes vary by vendor:

- **Brown/Black** = **GND**
- **Red** = **+5 V**
- **Orange/White** = **Signal (PWM)**

## Wiring

- Power the servo from a **separate 5 V supply** able to provide **≥1 A** surge per servo.  
- Tie **GND** of the servo supply and **ESP32 GND** together.  
- Connect **Signal** to any ESP32 GPIO that supports LEDC PWM (e.g., **GPIO 18**).

```
ESP32        SG92R
-----        -----
5V (ext)  -> +5V (red)
GND        -> GND (brown/black)
GPIO 18    -> Signal (orange/white)
```

## Gotchas

- **Do not** power from the ESP32 3.3 V pin. Use a **stable 5 V** source.  
- Provide **bulk decoupling** (≥470 µF) near the servo connector.  
- Avoid commanding beyond physical limits (~0–180°); excessive pulse widths can strip gears.  
- Expect **noise/EMI**; keep signal lead short or twisted with GND.

## How to use

**Arduino (ESP32):** Install the **ESP32Servo** library by Kevin Harrington (Library Manager).

```cpp
#include <ESP32Servo.h>

Servo s;
const int SERVO_PIN = 18; // any PWM-capable GPIO

void setup() {
  // Use a dedicated PWM timer/channel
  ESP32PWM::allocateTimer(0);
  s.setPeriodHertz(50);          // Standard 50Hz
  s.attach(SERVO_PIN, 500, 2400); // min/max pulse (us)
  s.write(90);                   // center
}

void loop() {
  for (int a = 0; a <= 180; a += 5) { s.write(a); delay(20); }
  for (int a = 180; a >= 0; a -= 5) { s.write(a); delay(20); }
}
```
